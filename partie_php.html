11. Transmettre des données entre pages

En PHP, il existe plusieurs façons de transmettre des données entre pages : par URL (méthode GET), par formulaire (méthode POST), ou via des sessions.

a. Par URL (méthode GET)

<!-- Lien HTML avec paramètres GET -->
<a href="page2.php?nom=Alice&age=25">Aller à la page 2</a>

<?php
    // page2.php
    if (isset($_GET['nom']) && isset($_GET['age'])) {
        $nom = htmlspecialchars($_GET['nom']);
        $age = intval($_GET['age']);
        echo "Bonjour $nom, tu as $age ans.";
    }
?>
Note : La méthode GET est visible dans l'URL et limitée en taille. Toujours valider et nettoyer les données reçues.

b. Par formulaire (méthode POST)

<!-- Formulaire HTML -->
<form method="post" action="traitement.php">
    <input type="text" name="nom" placeholder="Votre nom" required>
    <input type="email" name="email" placeholder="Votre email" required>
    <input type="submit" value="Envoyer">
</form>

<?php
    // traitement.php
    if ($_SERVER["REQUEST_METHOD"] == "POST") {
        $nom = htmlspecialchars($_POST['nom']);
        $email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL);
        echo "Bonjour $nom, votre email est $email.";
    }
?>
Note : La méthode POST est plus sécurisée pour les données sensibles, car elles ne sont pas visibles dans l'URL.

c. Par session

<?php
    // page1.php - Démarrer une session et stocker des données
    session_start();
    $_SESSION['nom'] = "Alice";
    $_SESSION['age'] = 25;
    echo "Session démarrée, données enregistrées.";

    // page2.php - Récupérer les données de session
    session_start();
    if (isset($_SESSION['nom']) && isset($_SESSION['age'])) {
        echo "Bonjour " . htmlspecialchars($_SESSION['nom']) .
             ", tu as " . intval($_SESSION['age']) . " ans.";
    }

    // Pour supprimer une variable de session
    unset($_SESSION['nom']);

    // Pour détruire toute la session
    session_destroy();
?>
Note : Les sessions permettent de conserver des données entre plusieurs pages pour un même utilisateur. Toujours démarrer une session avec session_start() avant d'y accéder.

11. Bonnes Pratiques

  • Utiliser des noms de variables et de fonctions explicites.
  • Séparer la logique PHP de la présentation HTML (utiliser des templates ou un framework comme Laravel).
  • Gérer les erreurs avec try/catch pour les exceptions.
  • Utiliser des commentaires pour expliquer le code complexe.
  • Ne pas afficher les erreurs PHP en production (utiliser error_reporting(0) et configurer php.ini).
